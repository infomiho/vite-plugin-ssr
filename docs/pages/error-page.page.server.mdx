When an error occurs (or when you call <Link href="/render" text={<code>throw render()</code>}></Link>), then the page `/renderer/_error.page.tsx` is rendered (if one is defined).

```js
// /renderer/_error.page.js

export { Page }

import { usePageContext } from 'vike-react'
/* Or:
import { usePageContext } from 'vike-vue'
import { usePageContext } from 'vike-solid'
*/

function Page() {
  const pageContext = usePageContext()
  let { is404, abortReason } = ctx
  if (!abortReason) {
    abortReason = is404 ? "This page doesn't exists." : 'Something went wrong. Sincere apologies. Try again (later).'
  }
  return <p>{abortReason}</p>
}
```

See <Link href="/usePageContext" />.

The page `_error.page.js` is rendered if:
 1. The URL didn't match the route of any of your pages (`404 Page Not Found`).
 2. One of your hooks (in `.page.js` / `.page.server.js` / `.page.route.js`) trhew an error (`500 Internal Error`).
 3. You called <Link href="/render" text={<code>throw render()</code>}></Link>.

> The error page is also rendered when vite-plugin-ssr throws an unexpected error (it has a bug), but since vite-plugin-ssr has no known bug this is very rare.


## Pre-rendering

If you use <Link text="pre-rendering" href="/pre-rendering" />, then vite-plugin-ssr uses `_error.page.js` to generate `/dist/client/404.html`.

> Most Static Hosts follow the convention that the file `404.html` is used as 404 page.
